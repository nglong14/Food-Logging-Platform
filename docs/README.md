## Conventions

- Documents are living artifacts — update them when design decisions change
- Major changes require a PR with doc updates alongside code
- Use diagrams (Mermaid or ASCII) for flows and architecture
- Mark sections as **Draft**, **Proposed**, or **Approved** as the project matures

## Status

All documents are in **Draft** status during the foundation phase.
